# Симуляція Мобільного Робота (Курсова Робота)

Проєкт реалізує 2D симуляцію руху мобільного робота в обмеженому середовищі з перешкодами.
Архітектура — **MVC**, рендеринг — **OpenGL 3.3 (core profile)**, інтерфейс користувача — **ImGui**.

Підтримується два режими роботи: ручне керування роботом та автономна навігація з побудовою
маршруту алгоритмом **A\***, а також повноцінний **редактор карти** для конструювання сцени.

---

## Функціонал

### 1. Фізика та колізії
- Рух робота керується напрямком та швидкістю (`velocity`), що задаються в реальному часі.
- Колізії обробляються методом **суб-степінгу** з бінарним пошуком точки контакту — це усуває
  «проскакування» крізь перешкоди на високій швидкості.
- Підтримуються типи колізій: **Circle–Circle**, **Circle–AABB (Rect)**, а також зіткнення
  з межами арени.
- Відображається точка зіткнення (Collision Point) та вектор поточної швидкості (Velocity Vector).

### 2. Сенсор Lidar
- Симуляція 2D-лідара з налаштовуваними параметрами: кількість променів, дальність (`max_distance`)
  та кут огляду (`FOV`, до 360°).
- Промені перевіряють перетин з усіма перешкодами та межами арени щокадрово.
- Лідар можна вмикати/вимикати окремо від інших налаштувань (`visible`).

### 3. Автономна навігація (Autopilot)
- Перемикається режим **Autonomous Mode**, у якому робот керується не клавіатурою, а цільовою точкою.
- Ціль задається **правим кліком** миші (у режимі симуляції) або **перетягуванням** хрестика цілі.
- Побудова маршруту — алгоритм **A\*** на квадратній сітці (8-напрямкова, без зрізання кутів
  при діагональному русі).
- **Два джерела карти прохідності (costmap):**
  - `Global Static Environment` — карта запікається один раз на основі реальних координат
    та розмірів перешкод зі сцени;
  - `Live Lidar Scan` — карта будується динамічно, накопичуючи точки зіткнення променів лідара
    в реальному часі (робот «дізнається» про перешкоду лише тоді, коли її «побачив»).
- Розмір клітинки сітки (`Cell Size`) налаштовується повзунком; для режиму Lidar є кнопка
  **Reset Lidar Map** для очищення накопиченої карти.
- Старт/Стоп руху по маршруту — кнопка в інтерфейсі або клавіша `Space`.
- Маршрут можна відображати трьома способами: **лінія**, **точки** або **квадрати**, з налаштуванням
  кольору та товщини.

### 4. Редактор карти (Map Editor)
Окремий режим програми для побудови сцени без перезапуску застосунку:
- **Select** — вибір та переміщення робота, арени чи перешкод (drag-and-drop), зміна розміру
  через gizmo-маркер у нижньому правому куті виділеної фігури.
- **Create** — створення нових перешкод (коло / прямокутник) кліком по робочій області з попереднім
  налаштуванням розміру, кольору заливки/обведення, режиму відображення (`Outline` / `Fill` /
  `FillAndOutline`) та товщини лінії.
- **Outliner** — список усіх об'єктів сцени (`System_Robot`, `System_Arena`, перешкоди) з можливістю
  вибору.
- **Inspector** — редагування трансформації (позиція, радіус/розміри) та зовнішнього вигляду обраного
  об'єкта.
- Робот та арена захищені від видалення (при спробі видалення підсвічуються червоним);
  довільні перешкоди видаляються клавішею `Delete` або кнопкою в інспекторі.
- Утримування `Ctrl` тимчасово перемикає на інструмент **Select** незалежно від поточного режиму.

### 5. Інтерфейс (ImGui)
- Панелі інтерфейсу можна вільно переміщати — вони автоматично обмежуються межами екрана.
- Перемикання режимів **Simulation / Map Editor** через меню або клавіші `1` / `2`.
- Інтерфейс можна повністю приховати клавішею `Tab` (наприклад, для скриншотів чи демонстрації).
- Пункт меню **Window → Reset Layout** повертає всі панелі у стандартні позиції.
- Меню **File**: відкрити конфіг у Notepad, зберегти поточну карту (`Ctrl+S`), перезавантажити сцену
  з файлу, зберегти/скинути користувацькі налаштування як дефолтні.

### 6. Конфігурація
Проєкт використовує три JSON-файли:
| Файл | Призначення |
|---|---|
| `config.json` | Геометрія сцени: розмір вікна, арена, перешкоди (тип, позиція, розмір, стиль), стартовий стан робота. Зберігається/завантажується через **Map Editor** і меню File. |
| `settings.json` | Налаштування поточної сесії: параметри лідара, видимість сітки/векторів, режим автопілота, стиль маршруту та цілі. Зберігається автоматично при закритті програми. |
| `settings_default.json` | «Заводські» налаштування для `settings.json`, на основі яких можна скинути сесію (`Reset to Defaults`) або які можна оновити поточними значеннями (`Save Current as Default`). |

## Керування

### Загальні
| Клавіша | Дія |
|---|---|
| `1` / `2` | Перемикання режиму: Simulation / Map Editor |
| `Tab` | Показати/приховати інтерфейс |
| `Ctrl + S` | Зберегти поточну карту у `config.json` |

### Режим Simulation
| Клавіша / дія | Дія |
|---|---|
| `W` `A` `S` `D` | Ручний рух робота (вимикається в Autonomous Mode) |
| `R` | Миттєве повернення робота на стартову позицію |
| ЛКМ (drag по хрестику цілі) | Перемістити цільову точку |
| ПКМ (в Autonomous Mode) | Встановити нову ціль у позиції курсора |
| `Space` | Старт / Стоп руху до цілі (тільки якщо ціль активна) |
| `Ctrl + Space` | Пауза / продовження фізики симуляції |

### Режим Map Editor
| Клавіша / дія | Дія |
|---|---|
| `S` | Інструмент Select |
| `C` | Інструмент Create (створення перешкод) |
| Утримання `Ctrl` | Тимчасовий перехід на Select незалежно від активного інструмента |
| ЛКМ | Вибір об'єкта / розміщення нової перешкоди |
| Drag по об'єкту | Переміщення обраного об'єкта |
| Drag по gizmo-маркеру | Зміна розміру обраного об'єкта |
| `Delete` | Видалити обрану перешкоду |
| `Ctrl + D` | Скинути виділення |

## Збірка з вихідного коду

### Вимоги
* **CMake** (версія 3.10 або вище)
* **MinGW-w64** (компілятор GCC/G++, підтримка C++17)
* **Git**
* Залежності у папці `dependencies/` (ImGui, GLM, glad) — мають бути присутні у репозиторії
  або підключені окремо

### 1. Клонування репозиторію
```bash
git clone https://github.com/TheFl00t/ProjectWork-Robot_movement_in_the_environment.git
cd ProjectWork-Robot_movement_in_the_environment
```

### 2. Підготовка
```bash
mkdir build
cd build
```

### 3. Компіляція
Доступні дві конфігурації: **Debug** (з консоллю) або **Release** (без консолі).

**Варіант А: RELEASE (рекомендовано)**
```bash
cmake -G "MinGW Makefiles" -DCMAKE_BUILD_TYPE=Release ..
cmake --build .
```

**Варіант Б: DEBUG**
```bash
cmake -G "MinGW Makefiles" -DCMAKE_BUILD_TYPE=Debug ..
cmake --build .
```

### 4. Запуск
Після збірки у папці `build` з'явиться `RobotSimulation.exe`. Система збірки автоматично копіює
поруч із виконуваним файлом `config.json` та папку `src/shaders`; файл `settings.json` буде
створено автоматично при першому запуску.
```bash
./RobotSimulation.exe
```
