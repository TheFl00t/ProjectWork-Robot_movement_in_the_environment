# Симуляція Мобільного Робота (Курсова Робота)

Проект реалізує 2D симуляцію руху робота в обмеженому середовищі з перешкодами.
Використовується архітектура MVC, OpenGL для рендерингу та ImGui для інтерфейсу користувача.

## Функціонал

1. **Фізика руху:** Робот рухається з заданою швидкістю, обробляються зіткнення (Circl-AABB, Circle-Circle).
2. **Візуалізація:** - Відображення перешкод (кола, прямокутники).
   - Відображення точки зіткнення.
   - Відображення вектора швидкості.
3. **Керування:**
   - `W`, `A`, `S`, `D` - рух робота.
   - `R` - миттєве повернення на стартову позицію.
   - `TAB` — скидання позиції вікна налаштувань.
4. **Конфігурація:** Всі параметри сцени завантажуються з файлу `config.cfg`.
5. **UI (ImGui):** Панель для зміни швидкості, радіусу та перезавантаження сцени у реальному часі.

## Збірка з вихідного коду

### Вимоги
Для успішної збірки на вашому комп'ютері мають бути встановлені:
* **CMake** (версія 3.10 або вище)
* **MinGW-w64** (компілятор GCC/G++)
* **Git**

### 1. Клонування репозиторію
```bash
git clone https://github.com/TheFl00t/ProjectWork-Robot_movement_in_the_environment.git
cd ProjectWork-Robot_movement_in_the_environment
```
### 2. Підготовка
Створіть папку для збірки:
```bash
mkdir build
cd build
```
### 3. Компіляція
Ви можете зібрати проєкт у двох конфігураціях: **Debug** (з консоллю) або **Release** (без консолі).
#### Варіант А: Збірка RELEASE (Рекомендовано)
Ця команда згенерує файли для MinGW та налаштує оптимізацію:
```bash
cmake -G "MinGW Makefiles" -DCMAKE_BUILD_TYPE=Release ..
cmake --build .
```
#### Варіант Б: Збірка DEBUG
Ця версія корисна, якщо ви хочете бачити вивід `std::cout` у консолі:
```bash
cmake -G "MinGW Makefiles" -DCMAKE_BUILD_TYPE=Debug ..
cmake --build .
```
### 4. Запуск
Після успішної збірки у папці `build` з'явиться файл `RobotSimulation.exe`. Система збірки автоматично копіює необхідні ресурси (`config.cfg` та папку `src/shaders`) поруч із виконуваним файлом.
```bash
./RobotSimulation.exe
```
